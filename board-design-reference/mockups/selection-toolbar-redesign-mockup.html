Notes & open questions

  • Unified prefix: every toolbar starts with the Color + Text pair (never separated — always the same two). Separators divide groups: Line groups Line style + Arrowheads; Section groups Re-scale + Border style, with Lock alone at the end — alternative would be Re-scale | Border style + Lock; confirm which grouping reads best.
  • Section "Rename" now uses the same T glyph as every other text control (was the A-in-brackets rename glyph).
  • "Change shape" now uses the dock's shapes icon (circle/square/triangle) instead of the swap-arrows icon, for consistency with the bottom toolbar.
  • Border style flyout dropped its text labels AND the "None" option (was Solid / Dashed / None with labels) — confirm "None" is really out. Trigger and options now use single-line solid/dashed glyphs instead of the stacked three-line icon.
  • Line toolbar gained a Text button (assumed: adds a midpoint label). Confirm this is the intent of "color, text, line style, arrow style".
  • Open-flyout trigger highlight is rgba(255,255,255,0.16); alternative is the purple active color — flip if it should read stronger.
  • Flyouts now center on their trigger and clamp to the viewport; previously they were pinned to the toolbar's left edge.
  • Color picker re-tuned to FigJam's measured recipe: 30px swatches, 12px gaps, 14px panel padding, hairline ring on every swatch, and a purple rounded-square outline on the current pick (replacing the rainbow ring — compare via the "Selected swatch" toggle).
  • FigJam's section toolbar also shows a shrink/resize control — decide separately whether we want one; not part of this pass.
  • Section gained a Scale-to-fit button (FigJam parity), placed right after Text, ordered by utility: Re-scale → Border style → Lock. In the demo it toggles the section hugging its title chip.
