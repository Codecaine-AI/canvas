Design / engineering reference

FigJam black pop-up toolbar — motion & elevation spec

A self-contained reference for reproducing FigJam's black selection-toolbar pop-up from frame-by-frame video analysis: live demos, measured data, and drop-in CSS.

TL;DR

What makes FigJam's pop-up feel "dynamic" and "floating" is three things working together:

  1. Entrance motion: it fades in while sliding up ~6px into place, ~140ms, strong ease-out. Never a plain instant show, never a plain fade.
  2. A hairline dark ring around the edge — reads as a crisp border and separates the black bar from anything behind it. It is not a CSS border; it is a zero-blur spread shadow.
  3. A two-layer drop shadow: a tight "key" shadow right under the bar plus a wide, faint "ambient" halo. The long soft tail is what sells the float.

Also: the exit is an instant cut (no fade-out), and sub-popovers (color palette) appear instantly — only the toolbar entrance animates. That asymmetry ("animate in, cut out") is part of why it feels snappy rather than laggy.

2. No slide — fade only

3. No ring

The black fill bleeds into the shadow; the edge stops feeling pixel-crisp on pale backgrounds.

4. Single flat shadow

The shadow looks blunt: grounded, but without the long faint tail that sells the float.

The Entrance Animation (Measured @ 30fps)

Properties animated: opacity 0 → 1 and transform: translateY(~+6px) → 0. No scale (the bar's width/edges are pixel-identical from the first visible frame), no horizontal motion.

Measured opacity ramp across consecutive frames (33ms apart): 0.65 → 0.86 → 0.96 → 1.00. i.e. ~65% complete within the first 33ms → strong deceleration curve.

Measured residual upward travel after the first visible frame: ~2.2px; extrapolating the same curve back to t=0 gives ~6px total travel.

Fully settled by the 4th frame → total duration ≈ 120–160ms (use 140ms).

Recommended easing: cubic-bezier(0.22, 1, 0.36, 1) (ease-out-quint-ish) or ease-out; must front-load ≥60% of progress in the first third.

The animation re-runs on every appearance: first selection, every selection change, and when the toolbar reappears after a drag ends (FigJam hides it during drag).

Exit: instant — full opacity to gone between two adjacent frames (≤33ms). Do not animate the exit.

Flyout/sub-popover (color palette row that opens above the bar): appears fully formed in a single frame. Do not animate flyouts.

Frame Time Measured opacity Motion note
Before appearance t=0ms 0.00 / gone Toolbar is not visible.
First visible frame t≈33ms 0.65 Already ~65% complete, with ~2.2px residual upward travel.
Second visible frame t≈66ms 0.86 Motion is decelerating strongly.
Third visible frame t≈99ms 0.96 Nearly settled.
Settled t≈132ms 1.00 Full opacity and position.

The Surface (Measured)

Fill #1D1D1D (spec as #1E1E1E, measured 29/29/29).
Radius Bar height in the capture: 37px; corner radius ≈ 10px (≈ 27% of height — scale radius with bar height, e.g. 40px bar → 11px radius; it is a rounded rect, NOT a full pill).
Glyphs Icon/glyph color ≈ #DCDCDC → spec rgba(255,255,255,0.85).
Divider Internal group divider: 1px vertical hairline, rgba(255,255,255,0.07), running nearly full height.

The bar floats ~8px above the selection bounds.

The Shadow System (Measured Luminance Falloff vs. Background)

Measured darkening of the background at distance from the bar's edge (background luminance 248):

Region Distance from edge Measured darkening
Below the bar 2px, 4px, 6px, 8px, 10px, 14px, 18px, 22px 2px→16.5%, 4px→12.5%, 6px→9.3%, 8px→6.9%, 10px→6.0%, 14px→4.4%, 18px→2.0%, 22px→0.8%
Above the bar 1px, 2px, 3px, 6px 1px→4.4%, 2px→1.6%, 3px→1.2%, 6px→0%
Sides Within 2–3px, then by ~10px ~6–7% within 2–3px, gone by ~10px

Interpretation: shadow is offset downward ~2px; a tight component (small blur) plus a wide component (~16–20px blur, low opacity) producing the long tail. Additionally the outermost 1px of the bar itself is darker than the fill (≈#161616 vs #1D1D1D fill) — a hairline ring, i.e. a zero-blur, 0.5px-spread black shadow at ~22–25% opacity. This ring is the "border" people sense but can't identify.

Recommended drop-in:

box-shadow:
  0 0 0 0.5px rgba(0, 0, 0, 0.22),   /* hairline ring — the "border" */
  0 2px 5px  rgba(0, 0, 0, 0.14),    /* key shadow — grounding */
  0 6px 18px rgba(0, 0, 0, 0.12);    /* ambient halo — the float */

Drop-in CSS

@keyframes selection-toolbar-in {
  from { opacity: 0; transform: translateY(6px); }
  to   { opacity: 1; transform: translateY(0); }
}

.selection-toolbar {
  height: 40px;
  background: #1E1E1E;
  border-radius: 11px;
  box-shadow:
    0 0 0 0.5px rgba(0, 0, 0, 0.22),
    0 2px 5px  rgba(0, 0, 0, 0.14),
    0 6px 18px rgba(0, 0, 0, 0.12);
  animation: selection-toolbar-in 140ms cubic-bezier(0.22, 1, 0.36, 1);
}

@media (prefers-reduced-motion: reduce) {
  .selection-toolbar { animation: none; }
}

Implementation notes

  • Restart the animation whenever the toolbar (re)appears or the selection changes — e.g. re-mount the element (change its React key) or re-trigger the animation.
  • No exit animation: unmount immediately.
  • Flyouts/popovers opened from the bar: no entrance animation.
  • Animate only opacity and transform (compositor-friendly); will-change: transform optional.
  • If the toolbar repositions while already visible (selection moved), FigJam does NOT replay the animation for pure moves of the same selection during e.g. zoom — replay only on appearance/selection change.
